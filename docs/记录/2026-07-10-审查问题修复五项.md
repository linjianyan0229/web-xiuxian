# 审查问题修复五项（私信入口/解散并发/登录态持久化/结交防重/丹药页溢出）

- 日期：2026-07-10 21:56
- 需求：代码审查发现 2 个 P1、3 个 P2 问题，逐项核实修复——非好友私信无阅读入口、传位与解散并发下旧宗主仍可解散、宗门操作后登录态未持久化、双向结交并发落两条反向行、丹药页手机端横向溢出。

## 改动内容

- 后端：
  - **[P1] 解散宗门并发守卫**：`sectModel.disbandSect(id, {leaderId})` 重排为「事务内先按 `DELETE FROM sects WHERE id=? AND leader_id=?` 带守卫删宗门行，0 行即回滚返 0，再清成员归属与成员表」；与 `transferLeader` 的 `UPDATE sects` 互斥于同一行锁，堵死「传位提交后旧宗主仍解散」的窗口。`doDisbandMySect` 传 `leaderId` 并对 0 行返 409「宗主之位已易主或宗门已散」；后台解散/删号级联不传守卫，行为不变。
  - **[P2] 结交请求并发防重**：`friendships` 表补**无向唯一键** `uk_pair_nodir((LEAST(requester_id,addressee_id)),(GREATEST(...)))`（DDL + 老库迁移 `ensureFriendshipPairIndex()`——先清既有反向重复[保留 id 较小者]再补键；需 MySQL ≥8.0.13 函数索引）。`relationController.requestFriend` 捕获 `ER_DUP_ENTRY` 后重查 `findBetween` 按实际关系回执 409。
- 前端：
  - **[P1] 非好友私信阅读入口**：`FriendsView` 好友页签下新增**「陌生传音」分组**（`conversations` 中不在好友集合的会话，带未读角标与最新一条摘要），点开即进右栏私聊可读可回；聊天区顶部「断交」按钮改为仅对好友显示（`isPeerFriend`）。
  - **[P2] 登录态持久化**：`stores/auth.js` 新增 `setUser(u)`（赋值 + `persist()`），替换全部 8 处 `auth.user = ...` 直赋值（SectView、SectDetailModal×3、MySectView×2、HomeView 头像回抛、AdminLayout）——修复立派/入宗后立即刷新 `/my-sect` 按 localStorage 旧 `sect_id` 误跳回宗门列表。
  - **[P2] 丹药页移动端溢出**：≤860px 媒体查询给 `.list-wrap` 加 `overflow-x:auto`、行加 `min-width:500px`（保证表头/行/底边线同宽）——列表在自身容器内横滚，页面 body 不再出横向滚动条（此前 375px 视口 scrollWidth=481px，操作列被裁出屏）。
