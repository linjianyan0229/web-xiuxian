# 宗门成员与职位系统(一期)

- 日期:2026-07-10 20:35
- 需求:按 `docs/设计文档/宗门机制.md` §8 待做清单与 `docs/设计文档/宗门职位与权限机制.md` 定稿,继续完善宗门功能——实装加入/退出、成员列表、职位体系基础与宗主前台管理(任免/逐出/传位/改资料/解散)。

## 改动内容

- 后端:
  - 新表 `sect_members`(id, sect_id, user_id 唯一, position, peak_id 预留, joined_time):记成员**职位**;归属仍以 `users.sect_id` 为准(人数聚合/连表不变),两处同事务写。启动时对存量入宗用户幂等回填(宗主→sect_master,其余→outer_disciple)。
  - 新工具 `utils/sectPositions.js`:14 职位枚举(key/name/rank/facilityLevel/quota),按文稿定职级与默认编制(太上长老 2、定额职位各 1、执法堂执事 3;峰主/真传弟子依赖山峰系统,一期不可任命;建宗自定编制待 `sect_quota` 表)。权限判定:人事权/信息权=rank≤3,长老级以上=rank≤5。
  - 宗门模型新增:`joinSect`(事务:锁宗门行防解散竞态 + 原子入宗守卫「无宗门+境界≥门槛」)、`removeMember`(退出/逐出共用,SQL 层拦宗主)、`appointMember`(事务:定额职位 FOR UPDATE 锁行校验编制)、`transferLeader`(事务:原宗主退居太上长老,编制校验)、`listSectMembers`(按职级 FIELD 排序)、`findMemberByUserId`。`createSect`/`disbandSect` 同步维护成员表。
  - 新接口(均需用户鉴权):
    - `GET /sects/:id/members` — 成员列表(分页,任何玩家可看;本宗成员附 `my` 职位与 `appointable` 可任命表)
    - `POST /sects/:id/join` — 拜入(境界门槛校验,入宗即外门弟子)/ `POST /sects/quit` — 退出(宗主 409)
    - `POST /sects/:id/appoint` — 任免(神子对长老级以上 403 须呈罢免信[文书系统二期];编制满 409)
    - `POST /sects/:id/kick` — 逐出(同人事权与神子限制)
    - `POST /sects/:id/transfer` — 传位(仅宗主;原宗主改任太上长老)
    - `PUT /sects/:id` — 高层修订资料(名称/简介/头像/背景/境界要求)/ `POST /sects/:id/disband` — 宗主解散
  - 详情接口 `GET /sects/:id` 增返 `my`(我在此宗的职位视图)。
  - 新日志类型:`sect_join/sect_quit/sect_kick/sect_appoint/sect_transfer/sect_disband`;`sect_appoint`、`sect_kick` 纳入通知 `NOTICE_TYPES`(被任免/被逐出者收到通知)。
  - 删号级联补 `sect_members` 清理。
- 前端:
  - 新组件 `SectMembersModal.vue`(门人名录):职位标签(长老级金色)/在线点/入宗时间;有人事权者行内「任免(下拉选职位含编制)/逐出」,宗主另有「传位」,均二次确认。
  - `SectDetailModal.vue` 重构:按身份呈现操作——散修「拜入此宗」(境界不足禁用+提示)、成员「退出宗门」、rank≤3「修订资料」、宗主「解散宗门」,所有人可看「门人名录」;入门要求行附「道友现居某位」。
  - `SectCreateModal.vue` 增编辑模式(mode='edit' 预填并走 PUT),复用为修订资料弹窗。
  - `SectView.vue` 详情弹窗传 meta、监听 `changed` 刷新;首页角色卡新增「宗门」行(sect_name,散修兜底)——文稿遗留项。
  - `api/game.js` 增 8 个接口函数;新 `utils/sectPositions.js` 前端职位映射(与后端对应)。
