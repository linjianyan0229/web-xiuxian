# 邮箱验证码（注册接码）与重置密码

- 日期：2026-07-08 01:40
- 需求：注册补上邮箱验证码（相关表、接口），邮箱 SMTP 配置写入 .env，写一个发信格式（邮件模板），并补上重置密码功能。

## 改动内容

- 后端：
  - **新表 `email_codes`**：`id, email, code(6位数字), purpose(register|reset), expires_at(+10分钟), used, created_time`，索引 `(email, purpose, id)`。一次一码：新码插新行，校验只认「最新一条未用未过期」，旧码自然作废。淘汰 `users.email_code` 预留列（dropColumn，注册时用户行尚不存在，独立表才成立）。
  - **SMTP 配置进 .env**：`SMTP_HOST/PORT/SECURE/USER/PASS/MAIL_FROM`（`.env.example` 附 QQ/163/Gmail 示例与授权码提示）。`SMTP_HOST` 留空视为未配置——**开发环境验证码打印到后端控制台**（本地联调无需真实邮箱），生产环境发送接口返回 503。
  - **发信服务与模板** `utils/mailer.js`（nodemailer ^7 新依赖）：`buildCodeMail()` 水墨仙侠风 HTML 模板（金印+大号验证码+有效期与防钓鱼提示，纯文本降级），`sendVerificationCode()` 发送。
  - **接口**（公开，挂 `/api/auth`）：
    - `POST /email-code`（`{ email, purpose }`）：注册要求邮箱未占用（409）、重置要求已立籍（404）；同邮箱同用途 60 秒限频（409）；返回 `{ ok, resendSeconds, ttlMinutes }`。
    - `POST /reset-password`（`{ email, emailCode, newPassword }`）：校验+核销验证码 → 换密码哈希 → **清空 access_token 吊销所有现有登录**，写日志 `reset_password`。
  - **注册接码**：`register` 在查重后校验 `emailCode`（`verifyAndConsume` 原子核销，防重放/并发复用）。系统配置 `register_email_code_enabled`（默认开）可临时关闭——未配 SMTP 的环境放行注册；重置密码不受开关影响。
- 前端：
  - 注册页加「邮箱验证码」字段 + 「获取验证码」按钮（60 秒倒计时，以服务端 `resendSeconds` 为准），emailCode 随注册提交。
  - 新页面 `/reset`（`ResetPasswordView.vue`，AuthShell 单栏）：邮箱 → 获取验证码 → 验证码+新密码+确认 → 成功 toast 并回登录页。登录页底部加「忘了密令？重置密码」入口。

## 验证

- 注册链路 curl 实测：发码 ok → 60 秒内重发 409 → 非法用途 400 → 无码注册 400 → 错码 400 → 正确码 201 → **验证码复用 400（单次使用）**。
- 重置链路 curl 实测：未注册邮箱 404 → 发重置码 ok → **拿注册码走重置（用途隔离）400** → 正确重置 ok → **旧令牌 401（已吊销）** → 旧密码登录 401 → 新密码登录成功；修行日志有 `reset_password` 记录。
- 前端 `npm run build` 通过。测试号「邮箱测试」（mailtest@test.local / newpass888）留在库中。

## 遗留

- 验证码枚举防护较薄：仅靠 10 分钟 TTL + 单次使用，未限制错误尝试次数——后续可加「同码错 5 次作废」或图形验证码。
- 发码接口无 IP 级限频（仅邮箱级 60 秒），公网部署建议在反代层加限流。
- 过期验证码行未清理（量小无碍），后续可在启动时顺手 `DELETE WHERE expires_at < NOW() - INTERVAL 7 DAY`。
- 真实 SMTP 发信未实测（本地无 SMTP 配置，走的控制台降级路径）；配置真实服务商后建议先用 `purpose=register` 发一封确认模板渲染效果。
