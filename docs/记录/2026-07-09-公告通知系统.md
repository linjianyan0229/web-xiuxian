# 公告与通知系统

- 日期：2026-07-09 00:52
- 需求：首页顶栏原「公告」按钮（✉）此前只是占位 toast。要求点击弹出小窗口，内分两类——「修仙公告」（管理员发布的全服公告）与「通知请求」（玩家收到的系统通知，如丹药赠礼）。按项目一贯做法做前后端全链路真实数据 + 后台管理页。

## 改动内容
- 后端：
  - 新增表 `announcements`（修仙公告：`id, title, content, pinned置顶, status(1发布/0下架), created_time, updated_time`，索引 `(status,pinned,id)`），在 `db.js` 建表清单中追加（幂等 CREATE IF NOT EXISTS）。
  - 玩家接口 `GET /api/user/announcements` → `{ announcements, notices }`：`announcements` 为已发布公告（置顶优先、再按 id 降序，取 30 条）；`notices` 复用 `player_logs`，只取系统通知类型（当前 `pill_receive`，收到丹药赠礼），取 30 条。
  - 后台接口：`GET /api/admin/announcements`（分页+标题关键字，含下架）、`POST`（发布）、`PUT /:id`（编辑）、`DELETE /:id`（删除）。
  - `playerLogModel` 新增 `listLogsByTypes(userId, types, limit)`（按类型 IN 筛选日志）。
- 前端：
  - 首页顶栏 ✉ 按钮改为「通知」入口，打开 `components/NotificationModal.vue`（水墨风分页弹窗：修仙公告 / 通知请求，各带条数角标）。公告项显示置顶徽记/标题/时间/正文；通知项显示叙事文本 + 时间。
  - 未读红点：以「最大公告 id : 最大通知 id」为签名，与 localStorage `noticeSeen` 比对；有更新则按钮右上角显示赤色小红点，打开弹窗即标记已读。进页 `loadNotices()` 背景拉取以驱动红点。
  - 后台新增「公告管理」页 `views/admin/AnnouncementsView.vue`（列表/搜索/分页 + 发布/编辑弹窗[标题/正文/置顶/状态] + 删除二次确认），路由 `admin-announcements`，侧边栏加入口（✉）。

## 验证
- 前端 `npm run build` 通过，生成 `AnnouncementsView` 分块。
- 后端另起 3100 端口实例连同库实测（不动 3000 现有服务）：
  - 管理员发布 3 条公告（含 1 条置顶、1 条下架）→ 后台列表返回 3 条，排序为 置顶→id 降序，`total=3`。
  - 玩家端 `/user/announcements` 只返回已发布 2 条且置顶在前，下架的不出现。
  - 编辑 id=2（改标题+下架）生效，玩家端随即只剩置顶 1 条。
  - 删除生效；空标题发布返回 400；玩家令牌访问后台接口返回 403。
  - 直插一条 `pill_receive` 与一条 `cultivate` 日志 → 玩家端 `notices` 只带出 `pill_receive`，`cultivate` 不混入。
- 实测后已清理测试玩家/日志/公告，临时 3100 实例已关闭。

## 遗留
- 「通知请求」目前仅纳入 `pill_receive`（收到丹药赠礼）。后续宗门邀请、好友申请、系统奖励等实装后，在 `announcementController.js` 的 `NOTICE_TYPES` 追加对应日志类型即可自动纳入。
- 公告未接敏感词检测（后台管理员发布，暂不强制）；如需可对 title/content 调 `containsSensitive`。
- 通知红点签名基于最大 id，弹窗打开时到达的新消息在关闭后会重新提示（可接受）。
