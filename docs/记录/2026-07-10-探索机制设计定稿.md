- 设计文档：`docs/设计文档/探索机制.md` 定稿到「可开工」程度。核心决定——
  - **形态**：挂机式，选时长档（沿用打坐 30/60/120/240 分钟），期满惰性结算；探索期间绝对互斥（锁死修炼/打坐/突破/再次探索）。
  - **区域**：不做静态区域表，改由 **DeepSeek 预生成缓存池**——后台按「境界段×难度」批量生成区域模板存池，玩家探索时服务端掷难度（简单80/困难15/危险5）后抽池填充；池空回退内置模板。境界合并为 **6 段** 分桶。
  - **防作弊红线**：难度服务端掷、收益服务端 clamp、出发即快照定死、AI 只管叙事骨架。
  - **收益**：灵石 + 丹药 + 材料 + 奇遇；丹药低阶掉成品、高难掉稀有材料（纯收集，需建材料表）；奇遇一期只发已有类型大额版；修为/道韵仅点缀。
  - **风险**：危险档小概率陨落（复用突破 death 范式，损失聚焦灵石）。
  - **加持**：白嫖可探，可选投灵石，加成 `60×S/(S+2B)` 饱和封顶 +60%。
  - **DeepSeek 配置**：后台绑 key + 选模型（系统配置项）。
  - **数值标定初稿**：§9.5 给了各境界段基础灵石/掉率/死亡率初值（可调）。
