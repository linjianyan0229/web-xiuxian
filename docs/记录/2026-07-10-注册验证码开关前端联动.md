# 注册验证码开关前端联动

- 日期：2026-07-10 20:02
- 需求：后台系统配置关闭 `register_email_code_enabled`（注册无需验证码）后，注册页仍显示验证码输入框和获取按钮，且前端本地校验强制要求填写，导致开关形同虚设。要求：开关关闭时注册页隐藏验证码行、跳过校验。

## 改动内容
- 后端：新增公开接口 `GET /api/auth/register-config` → `{ emailCodeEnabled }`，读系统配置 `register_email_code_enabled`（默认 true），供注册页判断是否需要验证码。注册接口本身逻辑不变（此前已支持开关关闭时跳过校验）。
- 前端：注册页 `onMounted` 拉取该配置存入 `codeRequired`（默认 true，**拉取失败保持显示验证码**，保守不误放行，最终判定仍在后端）；`codeRequired=false` 时——验证码 field 整行 `v-if` 隐藏、提交时跳过"请填写邮箱验证码"本地校验、注册 payload 不携带 `emailCode`。

## 涉及文件
- 修改：`node/src/controllers/authController.js`（新增 `registerConfig` 控制器）
- 修改：`node/src/routes/auth.js`（挂载 `GET /register-config` 公开路由）
- 修改：`web/src/api/auth.js`（新增 `apiRegisterConfig()`）
- 修改：`web/src/views/RegisterView.vue`（`codeRequired` 状态 + 验证码行 v-if + 提交逻辑分支）

## 验证
- 接口实测（本地起服）：
  - 开关=1：`GET /api/auth/register-config` 返回 `{"emailCodeEnabled":true}`；不带 emailCode 注册返回 400「请填写邮箱验证码」。
  - 开关=0（SQL 临时改库）：配置接口返回 `{"emailCodeEnabled":false}`；不带 emailCode 注册直接成功（201 返回 token+user）。
  - 测试后已恢复开关=1，并删除测试账号 `codetest01` 及其日志。
- 前端 `npm run build` 通过。

## 遗留
- 开关关闭时「获取验证码」发码接口（purpose=register）仍可调用，只是页面无入口；无安全影响（发码本身无副作用且限频），暂不额外拦截。
- 管理员在后台切换开关后，已打开的注册页需刷新才会感知（进页拉取一次，无轮询）；场景低频，暂不处理。
