# 审查问题修复五项 · 复审四项收尾（锁顺序/去重降级/拉黑会话/操作列吸附）

- 日期：2026-07-10 22:45
- 需求：上一轮五项修复的复审结论——无 P1，新增 3 个 P2、1 个 P3：传位与解散加锁顺序相反可能死锁、老库好友去重可能把已结交降级、已拉黑对象出现在「陌生传音」且可打开发送框、丹药页操作列藏在无提示横滚区。

## 改动内容

- 后端：
  - **[P2] 传位加锁顺序统一**：`sectModel.transferLeader` 改为**先 `SELECT ... FROM sects WHERE id=? AND leader_id=? FOR UPDATE` 锁宗门行并校验宗主**（不再靠尾部 UPDATE 判定），再做太上编制校验与成员职位变更，最后无条件 `UPDATE sects SET leader_id`。至此 joinSect / transferLeader / disbandSect 三个宗门事务加锁顺序统一为「先 sects 行、后 sect_members」，消除传位与解散并发反序互等的死锁窗口（全局错误处理无重试，死锁会直接 500）。
  - **[P2] 去重保留规则修正**：`ensureFriendshipPairIndex()` 清反向重复的 DELETE 条件由「保留 id 较小」改为「**status 高者优先**（已结交 > 待通过），同 status 保留 id 较小的先到者」——防止「小 id 是待通过、大 id 反向已结交」的老库把已结交关系降级回待应允。
- 前端：
  - **[P2] 陌生传音拉黑态**：`FriendsView` 新增 `blockedIdSet`/`isPeerBlocked`——分组内已拉黑对象名旁标「已拉黑」红标；打开其会话仍可读历史消息，但输入框与「传音」按钮禁用（占位文案「已拉黑此道友，解除拉黑后方可传音」，避免发送必 403 的死按钮），聊天区顶部「拉黑」按钮切为「解除拉黑」（复用既有 unblock 动作，成功后自动解禁）。
  - **[P3] 丹药页操作列吸附**：≤860px 时操作列（`.row > :last-child`，含表头「操作」）`position: sticky; right: 0` 吸附可视区右缘——不横滚也始终可见可点，不透明底遮住滚过的列、左缘投影示意有内容被压住；同时露出 6px 高的横向滚动条（`scrollbar-width: thin` + webkit thumb）作滚动提示。
