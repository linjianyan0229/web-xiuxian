# 丹药独立页面

- 日期：2026-07-10 20:21
- 需求：丹药从首页弹窗改为独立路由页（如宗门 `/sect`、伙伴 `/friends`）：① 列表展示，每行右侧带 服用/赠送/丢弃 三操作；② 筛选支持品阶（境界）、名称、品质；③ 展示丹药名称、数量、功效。

## 改动内容
- 后端：无（复用既有 `/api/user/pills`、`/api/user/pills/meta`、gift/discard 接口）。
- 前端：
  - 新增独立页 `views/PillsView.vue`（路由 `/pills`，requiresAuth）——套用宗门/伙伴页水墨壳（dongfu 背景、宣纸轻纱、锁一屏 + 列表内滚、窄屏放开整页滚动并隐藏功效/类型列）。列表列：品质标签/丹药名/品阶(境界)/类型/功效/数量 + 行内 服用/赠送/丢弃 按钮（`@click.stop` 不触发行点击）；功效列优先 `summary` 摘要，无摘要时由 `effects` 数组拼「目标+值%」简述。筛选：品质(凡/灵/道)、品阶(境界)、类型 下拉 + 名称关键字（下拉走 meta 接口仅含实际持有）；分页，变动后本页清空自动回退一页。
  - `PillDetailModal.vue` 新增 `initialMode` 属性（`main|gift|discard`，默认 main）：行内「赠送/丢弃」按钮直达对应面板，行点击进主面板看药力详情；服用仍为占位 toast（药力系统未接入）。
  - 路由 `router/index.js` 加 `/pills`；`App.vue` 的 `GAME_PAGES` 加 `pills`（获得游戏内页面方向滑动过渡）。
  - `HomeView.vue`：左侧导航「丹药」由开弹窗改为 `router.push({ name: 'pills' })`，移除 `PillBagModal` 引用与 `pillBagVisible` 状态。
  - 删除 `components/PillBagModal.vue`（已被独立页取代，删除前确认无残留引用）。

## 涉及文件
- 新增：`web/src/views/PillsView.vue`（丹药纳戒独立页）
- 修改：`web/src/components/PillDetailModal.vue`（新增 initialMode 直达面板）
- 修改：`web/src/router/index.js`（/pills 路由）
- 修改：`web/src/App.vue`（GAME_PAGES 加 pills）
- 修改：`web/src/views/HomeView.vue`（丹药导航跳页，移除弹窗）
- 删除：`web/src/components/PillBagModal.vue`

## 验证
- `grep -rn PillBagModal src/` 确认无残留引用后删除文件。
- 前端 `npm run build` 通过（PillsView 独立 chunk 正常产出）。
- 赠送/丢弃/筛选/分页逻辑均复用自原弹窗（已实测过的代码路径），页面壳复用宗门页样式。

## 遗留
- 「服用」仍为占位 toast，待药力/buff 系统接入。
- 页面未做浏览器端人工自查（本地无浏览器驱动），首次联调时留意窄屏栅格与行内按钮布局。
