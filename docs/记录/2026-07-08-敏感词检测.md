# 敏感词检测（词库合并 + 通用接口 + 世界频道拦截）

- 日期：2026-07-08 01:16
- 需求：根目录 `Vocabulary/` 下 17 个敏感词库文件，写一个检测接口供后续判断复用，世界频道现在就接上。

## 改动内容

- 后端：
  - **词库合并**：`Vocabulary/*.txt`（共 87044 行，全 UTF-8）合并去重为 `node/src/data/sensitive-words.txt`（50474 条，0.69MB，随库部署）——按「小写+全角转半角」归一化判重（35433 条重复），剔除单字条目 993 条（误伤率过高）、跳过 `A=B`/`A,B` 组合规则行 144 条（匹配器不支持）。根目录 `Vocabulary/` 为素材源，已加入 .gitignore。
  - **匹配器** `utils/sensitiveWords.js`：字典树（Trie），启动时 `initSensitiveWords()` 建树（76ms / 堆约 64MB / 200 字消息扫描约 0.03ms）。归一化口径同上（`ＦａＬunGong`≈`falungong`）；匹配途中允许跳过填充符，可识别「习*近*平」「法 轮 功」「兼*职」式变体（词条自身含符号的不受影响：先查树边，无边才跳）。导出 `scanSensitive(text)`（→ hit/words/sanitized 打码文本）与 `containsSensitive(text)`。
  - **白名单** `node/src/data/sensitive-words-allow.txt`：剔除词库中误伤游戏正常交流的独立词条——摸底 60+ 游戏常用词后剔除 8 条：修炼/弟子/师父/功法/大法（修仙核心词）+ 测试/联系/网络（日常词，源词库反广告过宽）。剔除独立词条不影响完整词条（「大法」剔除后「法轮大法好」仍命中）。运维直接编辑两份 txt、重启生效。
  - **通用接口** `POST /api/user/sensitive/check`（`{ text }` ≤5000 字）→ `{ hit, words, sanitized }`，供前端预检与后续玩法（道号/宗门名/简介等）复用。
  - **世界频道拦截**：`postWorldMessage` 在长度校验后、冷却校验前调 `containsSensitive`，命中返 400「言辞含违禁之语…」（改词重发不吃冷却）。
  - `server.js` 启动序列加 `await initSensitiveWords()`。
- 前端：无（聊天 400 文案经既有 toast 展示）。

- 修改：`node/src/controllers/worldChatController.js`（发言拦截）、`node/src/routes/user.js`（挂路由）、`node/src/server.js`（启动加载）、`.gitignore`（Vocabulary/）

## 验证

- 离线：归一化/变体/白名单用例全过——「今日打坐修炼收获颇丰」「师父带你修功法」「太上大法真妙」通过；「法轮大法好」「法 轮 功」「兼职」命中。
- HTTP 实测：check 接口命中（words+打码）/干净/空文本 400；聊天「兼*职」变体 400、「今日修炼小有所成」正常 201（测试消息已清理）。

## 遗留

- 词库仍偏宽泛（如「弟子规」会因含「弟子」被白名单放行后单独判定——目前无此词条，无碍），后续遇误伤直接往 allow 文件加词即可；遇漏拦往主词库补。
- 检测目前只接了世界频道；注册道号、宗门名称/简介、赠丹留言等入口建议后续统一接入（调 `containsSensitive` 即可）。
- 词库更新流程：改 `Vocabulary/` 源文件后重跑合并脚本（scratchpad 一次性脚本，规则见 sensitive-words.txt 头部注释）；或小改动直接编辑合并产物。
- 内存占用约 64MB（5 万词 Map 树），如未来词库翻倍可考虑双数组 Trie/Aho-Corasick 压缩。
