# 好友/私信/拉黑社交系统

- 日期：2026-07-09 01:09
- 需求：世界频道用户名片（点击道友弹出）右侧信息栏新增「结交、切磋、私信、拉黑」四个操作按钮；后端实装结交（加好友）、私信、拉黑功能，切磋暂只做交互按钮占位。

## 改动内容
- 后端：
  - 新增三张表：`friendships`（好友关系/请求，status 0=待通过/1=已结交，方向记 requester/addressee）、`user_blocks`（单向拉黑）、`private_messages`（点对点私信，receiver_read 已读标志）。均在 `db.js` 建表并纳入 `initDatabase`；`deleteUser` 手动级联清理三表。
  - 新增关系/私信接口（均需用户鉴权）：
    - 关系状态 `GET /user/relations/:userId`（名片按钮态判定）
    - 好友 `GET /user/friends`、`POST /user/friends/request|accept|reject|remove`
    - 拉黑 `GET /user/blocks`、`POST /user/blocks`、`POST /user/blocks/remove`
    - 私信 `GET /user/messages`（会话列表+未读）、`GET /user/messages/unread`、`GET /user/messages/:userId`（取会话并标记已读）、`POST /user/messages`（发送）
  - 规则：结交/私信前双向拉黑校验（任一方拉黑即拦截）；拉黑即断交（清除既有好友/待通过请求）；私信复用敏感词拦截 + 控制字符清洗 + ≤200 字；结交请求/应允写系统通知（`player_logs` type=`friend_request`/`friend_accepted`，已纳入通知弹窗 NOTICE_TYPES）。
- 前端：
  - `ChatUserModal.vue`：右侧信息栏加四按钮，打开时按 `user_id` 拉取关系状态（`apiRelation`），结交/拉黑按钮文案随关系动态切换（结交/接受结交/待通过/已是道友；拉黑/解除拉黑），私信在拉黑关系下禁用，切磋为占位 toast。
  - 新增 `PrivateChatModal.vue`（一对一私信会话弹窗，5 秒轮询、气泡区分己方、发送标记已读）。
  - **新增独立路由页 `views/FriendsView.vue`（`/friends`，名「道友往来」）**：水墨风壳（复用 dongfu 背景、锁一屏内滚，同宗门页），**经典 IM 双栏布局**——左栏列表（好友[在线态+未读角标] / 群聊[占位] / 结交请求 / 黑名单 四页签），右栏为**内嵌私聊区**（点好友即载入会话：消息滚动区 + 顶部对方信息与断交/拉黑 + 底部发送区，5 秒轮询、进会话标记已读），未选会话显示引导空态。已加入路由与 `App.vue` 的 `GAME_PAGES`（获得滑动过渡）。（`PrivateChatModal` 保留，仅供世界频道名片「私信」按钮使用。）
  - `HomeView.vue`：左侧导航「伙伴」跳转 `/friends`；频道名片「私信」打开 PrivateChatModal；「伙伴」导航项加红点角标（未读私信数 + 待处理结交请求数，进页拉取并每 30 秒背景刷新）。
  - `api/game.js` 补齐 relations/friends/blocks/messages 系列封装。

- 好友暂无数量上限与备注/分组；黑名单不影响世界频道可见性（仅拦截结交/私信）。
