# 时间展示时区修复（统一按 Asia/Shanghai）

- 日期：2026-07-08 01:26
- 需求：世界频道聊天时间不对，没有按亚洲上海时区显示。

## 改动内容
- 后端：无（服务器/MySQL 时钟为 UTC，接口返回带 Z 的 ISO 串，时刻本身正确）。
- 前端：根因是各页面的 `fmt()` 直接对 ISO 串 `replace('T',' ')` 截取——把 UTC 钟面原样显示，比北京时间慢 8 小时。新建共享工具 `web/src/utils/datetime.js`：`fmtDateTime()` 用 `Intl.DateTimeFormat`（en-CA + hourCycle h23）**固定按 Asia/Shanghai** 输出 `YYYY-MM-DD HH:MM:SS`（不随访问端系统时区漂移），空值/无效返回 '—'。替换全部 6 处截串格式化：HomeView（世界频道/修行日志/角色卡入道与上次登录）、AttributeModal（上次登录）、admin/UsersView（注册/登录时间）、admin/SectsView、SectView（创建日期）、SectDetailModal。各处原有 slice(5,16)/slice(0,10) 二次截取按新格式仍成立。

## 涉及文件
- 新增：`web/src/utils/datetime.js`
- 修改：`web/src/views/HomeView.vue`、`web/src/components/AttributeModal.vue`、`web/src/views/admin/UsersView.vue`、`web/src/views/admin/SectsView.vue`、`web/src/views/SectView.vue`、`web/src/components/SectDetailModal.vue`

## 验证
- node 实测格式化：UTC `17:49Z` → `2026-07-08 01:49:17`（+8 正确、跨午夜日期正确），slice 派生格式（MM-DD HH:MM / 日期）正常，空值/无效值回退 '—'；前端构建通过。

## 遗留
- 倒计时类逻辑（修炼冷却/打坐剩余）走 `new Date()` 时刻差值，本就与时区无关，未动；新增时间展示请统一走 `utils/datetime.js`，勿再截 ISO 串（已在 CLAUDE.md 注明）。
