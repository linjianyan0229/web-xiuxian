# 成丹材料与丹方机制（数据初稿）

> **状态：种子数据 + 建表 seed + 只读接口已实装（2026-07-11）**——材料库 748 种 + 丹方 435 张已产出到 `node/src/data/pill-materials.json`，并落库三表（`pill_materials`/`pill_recipes`/`pill_recipe_items`，启动幂等导入）；后台材料/丹方查询接口与玩家丹方查阅接口已上线（见 §6）。**炼丹玩法、材料背包、材料获得途径未实装**。
> **关联文稿**：丹药数据见 `资源文件/丹药相关/pills.md`；服用/药力系统未实装（见 CLAUDE.md 丹药背包条目）。

## 1. 总则

- 每种丹药对应**一张丹方**（配方），列出炼制所需材料与份数。
- **仙王及以上（仙王/仙帝/道祖/圣人，即「圣王以上」）共 40 种丹药不设丹方**——无法炼制，只能由机缘获得（对应 pills.json 的 `immortalKingAndAbove`）。
- 其余 **435 种丹药**（凡人~大罗金仙）每丹一方，**少则 5 味、多则 20 味**，味数随境界档递增。
- 同一丹方可炼凡/灵/道三品；**成丹品质由炼丹系统结算**（灵/道品可要求材料份数倍率，实装时定）。

## 2. 材料模型

材料按 **22 个大境界档** 分层（凡人~大罗金仙，与丹药境界对齐），每档 34 种、全库 **748 种**。

| 维度 | 取值 | 说明 |
| --- | --- | --- |
| 类型 `type` | herb 灵植 / mineral 矿石 / beast 兽材 / essence 精魄 / misc 丹引 | 每档配比 12/7/7/5/3 |
| 稀有度 `rarity` | common 凡品 / uncommon 良品 / rare 上品 / epic 珍品 / legendary 绝品 | 权重 40/30/18/9/3%；价格乘数 1/2.5/6/15/40 |
| 价格 `price` | `round(5 × 1.55^档位 × 稀有度乘数 × 0.85~1.20 抖动)` 灵石 | 凡人档个位数~几十，大罗绝品可达百万级 |
| 产出地 `origins` | 每材料 1~3 处，按界域分池 | 凡俗（凡人~心动）/ 尘世（金丹~渡劫）/ 仙家（地仙~大罗）各 10 处 |
| 境界 `realm/realmRank/minRealmId` | 大境界名 + 大境界序号（与 pills.json 的 realmRank / `pills.realm_rank` 同语义，凡人=1~大罗金仙=22）+ 该大境界最小 realms.id（与 `users.realm_id` 比较的门槛用） | 丹方取本档为主、可掺低一档材料 |

- **兜底规则**：每档每类型首个材料强制「上品」（保证主药可选）；每档首个丹引强制「凡品」（保证有廉价引子）。
- 名称由「界域前缀 × 类型主体」组合生成，全库唯一；描述为模板句（产地+稀有度+类型口吻）。

## 3. 丹方规则

- **味数**：`5 + round(档位×13/21) + 0~2 抖动`，收敛 [5,20]——凡人 5~7 味、大罗金仙 18~20 味。
- **结构**：每方固定 = **主药 1 味 + 辅药 n−2 味 + 丹引 1 味**（`role: main/aux/catalyst`）。
  - **主药**：本档、上品及以上，类型呼应丹药类别——修为/疗伤/悟性丹→灵植，攻击丹→兽材，防御丹→矿石，精神/灵气/突破/免死/道韵/道法丹→精魄，毒丹（down 类）→兽材为主。
  - **辅药**：从本档 + 低一档的非丹引材料中不重复抽取，份数 2~9。
  - **丹引**：本档 misc 类一味，份数 1~2；主药份数 1~3。
- 生成为**确定性随机**（mulberry32，种子 20260711），改配置重跑 `资源文件/丹药相关/generate-materials.mjs` 即全量重建。

## 4. 数据文件

- **生成器**：`资源文件/丹药相关/generate-materials.mjs`（运行 `node generate-materials.mjs`）。
- **产物**：`node/src/data/pill-materials.json`，顶层 `{ version, title, rules, materialTypes, rarities, originZones, stats, materials[748], recipes[435] }`。
  - 材料：`{ id(mat_<境界拼音>_<序号>), name, type/typeName, rarity/rarityName, realm/realmRank/minRealmId, price, origins[], description }`。
  - 丹方：`{ pillId, pillName, realm, realmRank, category, materialCount, materials:[{materialId, name, quantity, role}] }`。
- 生成器自带校验：味数 5~20、材料引用存在、单方材料不重复、材料名全库唯一、材料档 realmRank 与 pills.json 同名境界序号一致。

## 5. 表结构与后端接入（已实装）

- **三表**（DDL 内联 `node/src/config/db.js`，无外键）：
  - `pill_materials`：字符串主键 + 名称唯一键，`type/type_name、rarity/rarity_name、realm/realm_rank(1~22)/min_realm_id、price、origins(JSON)、description`，索引 `realm_rank/type/rarity`。
  - `pill_recipes`：`pill_id` 主键 + `material_count`（丹方头，预留成功率/耗时等扩展字段位）。
  - `pill_recipe_items`：一行一味，唯一键 `(pill_id, material_id)`，`quantity、role(main/aux/catalyst)、sort_order`，索引 `material_id`。
- **seed**：`seedPillMaterials()`——材料判本表非空跳过；丹方**判主表与明细表任一为空即事务内清残局重导**（防主表提交后进程中断留下「有丹方无材料」的半截状态；材料 748 / 丹方 435 / 明细 5530），启动时在 `seedPills()` 之后执行。
- **数据访问** `models/pillMaterialModel.js`：`materialMeta`/`listMaterials`（分页+类型/稀有度/境界/关键字筛选）/`findMaterialById`/`findRecipeByPillId`（明细连材料表，主药→辅药→丹引排序）/`listRecipes`（连丹药名）。
- **控制器** `controllers/pillMaterialController.js`（当前全只读）。

## 6. 接口一览

玩家（需用户鉴权）：
- `GET /api/user/pills/:pillId/recipe` — 丹方查阅（图鉴/炼丹预览）→ `{ recipe }`（含 items 材料明细：名称/类型/稀有度/价格/产地/份数/角色）；丹药不存在或圣王以上无方 404。

后台（需管理员鉴权）：
- `GET /api/admin/materials/meta` → `{ types, rarities, realms }`（筛选下拉；先于 `:id` 声明）
- `GET /api/admin/materials?page&pageSize&keyword&type&rarity&realm` → `{ list, total, page, pageSize }`
- `GET /api/admin/materials/:id` → `{ material }`；不存在 404
- `GET /api/admin/recipes?page&pageSize&keyword&realm&category` → 丹方列表（连丹药名/境界/类型）
- `GET /api/admin/pills/:id/recipe` → `{ recipe }` 丹方详情；无方 404
