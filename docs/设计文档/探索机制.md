
# 探索机制（草案 · 讨论稿）

> **状态：草案，未实装。** 本文用于「边聊边完善」——先铺骨架、标清待定项，数值/规则随讨论迭代。
>
> **已敲定的骨架决定（2026-07-09 讨论）：**
>
> - **形态：挂机式**——选区域后进入历练倒计时，期满一次性惰性结算（复用打坐范式）。
> - **区域来源：DeepSeek 大模型动态生成**——不做静态区域表，出发时由服务端先定难度、再让大模型按角色属性即时生成区域（地名/描述/事件/见闻）。**难度分布：简单 80% / 困难 15% / 危险 5%**。
> - **难度—收益—风险正相关**：难度越高，收益越高、危险也越高（危险档小概率陨落）。
> - **时长档位：沿用打坐的 30/60/120/240 分钟**四档。
> - **绝对互斥**：探索期间**不允许执行任何其他动作**（修炼、打坐、突破、再次探索全部拦截）。
> - **风险：分级，危险档可陨落**——简单/困难档无致命惩罚，危险档小概率陨落（复用突破死亡范式：`death_count+1`、资源折半）。
> - **收益重心：灵石 + 丹药 + 奇遇叙事**——修为/道韵仅作小额点缀，不作主线。
> - **系统定名：探索**——前台入口按钮即「探索」。
> - **丹药产出：两者都掉**——低阶/简单掉成品丹（直接进背包），高阶/高难掉稀有炼丹材料（**纯收集**，为炼丹系统埋钩子，需建材料表）。
> - **前置消耗：可选加持**——白嫖即可探索；**可选投入灵石，投入越多收益越多**（风险投资，投入不退）。
> - **奇遇产物：一期先只发已有类型**——奇遇先发灵石/丹药/材料的「大额版」；残卷/秘籍/称号等新实物待对应系统落地再接。
> - **战斗替代**：`enemy` 事件在战斗系统落地前，用纯掷骰替代（认可）。
> - **生成方式：预生成缓存池**——不每次实时调大模型；后台预先批量生成一批区域模板存池，玩家探索时抽取填充，控成本降延迟（详见 §3）。
> - **境界段：6 段分桶**（凡俗/筑基/元婴/大乘/仙人/仙王，详见 §3.6）。
> - **DeepSeek 配置：后台绑定**——API Key 与模型在后台「系统配置」里绑定/选择，不写死。
> - **专属感：模板占位符替换**——抽到模板后把道号/宗门/境界填入占位符，轻量增强代入感（不额外调 AI）。
> - **前端流程：先投入后揭晓**——玩家先选时长+可选加持灵石再「出发」，难度与区域出发后才揭晓（保悬念、防按难度挑区域破坏 80/15/5 分布）。
> - **数值：见 §9.5 标定初稿**（收益/加持/死亡率均给了可调初值）。
>
> **一句话定位：** 玩家离开洞府外出「探索」由大模型即时生成的秘境与险地，投入时间（可选加持灵石、承担风险）换取灵石、丹药、材料与奇遇。
> **与现有系统的关系：** 探索是**新的资源产出与消耗闭环**，串联境界（影响生成与收益）、丹药/材料（掉落）、宗门（后续可扩展）。修炼/打坐是「闭门精进」，探索是「外出求机缘」，二者互补且互斥（同一时间只能做一件）。

## 1. 玩法定位与设计目标

- **给玩家一个「主动外出」的资源入口**：目前修为靠修炼/打坐、灵石靠签到，缺少「冒险博收益」的玩法。探索填这个空。
- **引入风险—回报权衡**：与打坐「稳定挂机」不同，探索可以有失败/受伤/空手而归，甚至（高危区域）陨落风险，收益上限更高、方差更大。
- **承载随机性与叙事**：随机事件/奇遇（遇宝、遇敌、遇前辈、遇秘境）给文字修仙的「叙事感」提供载体，也是后续埋战斗、炼丹材料、功法残卷等系统的钩子。
- **挂机形态**：像打坐一样「派出去、到点回收」，期满惰性结算（见 §7）。

## 2. 核心玩法循环（已定）

```
点「探索」 → 服务端掷难度(简单80/困难15/危险5) → DeepSeek 按角色属性生成区域内容
   → 玩家选时长档(30/60/120/240)、可选加持灵石 → 出发，进入历练倒计时（期间锁死其他动作）
   → 期满惰性结算 → 掷事件/收益/风险 → 发放资源·丹药·材料·奇遇物品 → 写日志/今日统计
```

- **入口**：前台首页左侧模块导航新增「探索」按钮，打开探索弹窗/独立页。
- **一次探索 = 一段时长**：类比打坐的定时挂机，出发后进入「探索中」倒计时，期满惰性结算（沿用打坐范式，无需定时任务）。
- **不可中断**：出发后无取消/召回接口，期满才结算（同打坐）。

## 3. 区域生成（已定：DeepSeek 生成 + 预生成缓存池）

**不做静态区域表，也不每次实时调大模型**，而是「**后台预生成一批区域模板存池 → 玩家探索时抽取填充**」：

**A. 离线预生成（后台/定时批量）**

- 用 DeepSeek 按**难度档 + 境界段**批量生成区域模板，写入区域池表（见 §9），每条含：地名、场景描述、氛围文案、事件序列骨架、见闻文本。
- 池子按「难度 × 境界段」分桶，保证各境界段各难度都有货；池水位低时补生成。
- **内容只含文案与事件类型，不含最终数值**（数值恒由结算时服务端定）。
- 大模型产出入池前过敏感词检测与 JSON 结构校验，不合规丢弃。

**B. 玩家点「探索」（实时、无大模型调用）**

1. **服务端掷难度**（固定分布，客户端不可干预）：**简单 80% / 困难 15% / 危险 5%**。
2. 按「玩家当前境界段 × 已掷难度」从池中**抽取一条区域模板**（抽中可标记降权/短期不重复，避免连刷同一区域）。
3. 若对应桶恰好空（预生成没跟上）→ **回退内置模板区域**，探索照常进行（不卡死）。
4. 出发时把「难度 + 区域模板 + 时长 + 加持灵石 + 预结算收益种子」快照进 `explore_snapshot`（见 §9）。

> **权衡（记录）**：预生成池换来低成本、低延迟、可控内容，代价是区域**不再是「为当前角色实时定制」**，而是「按境界段+难度匹配」的近似定制。如需更强的「专属感」，可在抽取后用轻量变量替换（把道号/宗门名等填入模板占位符），不额外调大模型。是否要这层替换见 §10。

## 3.5 防作弊 / 稳定性红线（AI 生成带来的新约束）

- **服务端权威**：所有资源/丹药/材料/奇遇的**最终数值与是否发放，全部由服务端结算**，大模型输出仅作叙事与事件骨架；任何数值一律 clamp 到「难度 × 境界」允许区间。（对齐 CLAUDE.md「核心数值由后端负责，防客户端篡改」。）
- **难度服务端掷定**：80/15/5 的难度在服务端产生，不进大模型的「自由裁量」，避免被提示词绕过刷高难度收益。
- **失败降级**：大模型超时/报错/返回不合规 JSON 时，回退到一套**内置模板区域**（保底文案 + 正常结算），探索不因 AI 不可用而卡死。
- **成本与延迟**：**大模型调用发生在后台预生成阶段，与玩家点探索解耦**——玩家侧只做「抽池 + 结算」，无实时大模型延迟；预生成可批量/定时/低峰跑。
- **内容安全**：大模型产出的文案需过既有敏感词检测（`containsSensitive`）后再入池/展示。

## 3.6 境界段分桶（已定：6 段）

26 个大境界太碎，合并为 **6 段**用于区域池分桶与收益标定（按 `realms.id` 区间归段）：

| 段     | 代号           | 含大境界                                         | realm_id 区间 | 首段 advance_exp 量级     |
| ------ | -------------- | ------------------------------------------------ | ------------- | ------------------------- |
| 1 凡俗 | `mortal`     | 凡人·炼体·练气                                 | 1–9          | 80 ~ 220                  |
| 2 筑基 | `foundation` | 筑基·开光·融合·心动·金丹                     | 10–29        | 480 ~ 11500               |
| 3 元婴 | `nascent`    | 元婴·出窍·分神·合体                           | 30–45        | 25500 ~ 280000            |
| 4 大乘 | `mahayana`   | 洞虚·大乘·渡劫                                 | 46–57        | 615000 ~ 3000000          |
| 5 仙人 | `immortal`   | 地仙·天仙·真仙·玄仙·金仙·太乙金仙·大罗金仙 | 58–79        | advance_exp=0（道韵晋级） |
| 6 仙王 | `sovereign`  | 仙王·仙帝·道祖·圣人                           | 80–83        | advance_exp=0（道法晋级） |

- 归段函数按 `realm_id` 落桶（服务端 `utils/explore.js` 内一个区间映射）。
- 区域池按 `(境界段 × 难度)` = 6 × 3 = **18 桶**存模板，各桶维持一定水位。
- 仙人/仙王段 `advance_exp=0`，其修为点缀改折算为**道韵**（极小额），主收益仍是灵石/材料。

## 4. 收益类型（初稿）

一次探索的收益由「基础产出 + 随机事件产出」组成。**收益重心（已定）：灵石 + 丹药 + 奇遇叙事**，修为/道韵仅点缀：

- **灵石（重心）**：主力产出，给灵石一个持续水龙头（当前灵石偏静态）。
- **丹药（重心）**：**分级产出**——低阶区域直接掉成品丹（进 `user_pills` 背包），这将是**玩家获得成品丹药的首个正式途径**（目前背包无正式产出）；掉率/品质保守起步。
- **炼丹材料（重心/高难）**：高难度区域掉稀有炼丹材料——**纯收集**（炼丹系统落地前只作为可收集/展示品，不可兑换、不悬空），需**新建材料表**（`materials` / `user_materials`）。
- **奇遇 / 机缘（重心，一期简化）**：小概率大收益。**一期只发已有类型的「大额版」**（成倍灵石 / 高品质丹药 / 稀有材料），走已有的入库通道；**残卷/秘籍/称号等新实物待对应系统落地再接**（见 §10）。奇遇的「惊喜感」一期靠数值量级 + AI 见闻文案营造。
- **修为 / 道韵 / 道法（点缀）**：仅小额补充，占比刻意压低，不抢修炼/打坐主线。

> 收益规模与「难度 × 境界」挂钩：难度越高、境界越高，收益区间越大；服务端锚定 `advance_exp` 与打坐/签到量级标定，避免破坏经济平衡（见 §8）。

## 4.5 可选加持（已定：投灵石越多收益越多，饱和曲线封顶）

探索**不强制前置消耗**——纯投入时间即可出发。玩家可在出发前**可选投入灵石**加持本次探索（风险投资，投入不退）：

- **加持公式（初稿，饱和递减 + 封顶）**：设本次「基础灵石收益」为 `B`，玩家投入 `S` 灵石，则本次收益总加成为
  ```
  加成% = 60 × S / (S + 2B)
  ```

  即 `S=2B` 时约 +30%、`S=6B` 时约 +45%、`S→∞` 趋近 **+60% 封顶**。天然边际递减，堆再多也吃不满。
- 加成作用于**灵石收益 + 丹药/材料掉落权重**（不作用于陨落率；加持提升的是回报，不买命）。
- 投入在**出发时**锁定扣除（事务内，仿开宗立派扣灵石），中途不可追加。
- **失败/空手/陨落时投入不退**（体现风险）；陨落时加持灵石随本次收益一并损失（见 §5）。
- 投丹药加持：暂不做（先只做灵石），后续扩展。

## 5. 风险机制（已定：按难度档，危险档可陨落）

风险跟随 §3 掷定的**难度档**，不再单列区域：

| 难度 | 占比 | 惩罚形态                                                                                                           |
| ---- | ---: | ------------------------------------------------------------------------------------------------------------------ |
| 简单 |  80% | 无致命惩罚，最多「所获平平」（收益偏下限）                                                                         |
| 困难 |  15% | 可能空手 / 资源折损（损失部分本次收益或少量加持灵石）                                                              |
| 危险 |   5% | 收益最高，但**小概率陨落**：复用突破死亡范式——`death_count+1`、对应资源折半、写 `death` 日志、境界不跌 |

- **陨落率（初稿）**：仅危险档触发，**基础 8%**；随境界段在 **6%~12%** 微调（越高段的危险区越凶，服务端掷定，可由系统配置改）。
- **陨落惩罚（初稿）**：`death_count+1`、境界不跌、写 `death` 日志；损失=**当前灵石折半 + 本次全部已得收益与加持灵石作废**（复用突破 death 范式的「资源折半」，探索折灵石这一主货币；修为不动，惩罚聚焦灵石更契合探索定位）。
- **困难档折损（初稿）**：约 30% 概率「空手/折损」，折损=本次已得收益打 5 折 + 加持灵石损失一半（不碰身家灵石）。
- **受伤/气血**这类需新属性的软惩罚**暂不引入**（当前无 HP 玩法），先用「空手/折损/陨落」表达，等战斗/气血系统落地再补。
- 结算一律在后端（防篡改），原子 UPDATE 带守卫（同突破/打坐）。

## 6. 随机事件系统（初稿）

一次探索期间产生 N 个「事件」（N 随时长/难度）。**事件类型骨架由服务端定义**，DeepSeek 负责为每个事件生成贴合区域的**见闻文案**，事件的**收益/风险数值仍由服务端结算**。示意事件类型：

- `treasure` 寻获灵石/丹药/材料
- `herb` 采得草药（材料）
- `enemy` 遭遇妖兽/修士 → 战斗结算（战斗系统未做，**暂以掷骰替代**，已认可）
- `senior` 偶遇前辈 → 传功/赠物（小概率大收益）
- `ruin` 发现小秘境 → 额外收益
- `trap` 陷阱/心魔 → 折损（危险档可能触发陨落判定）
- `fortune` 奇遇 → 一期发已有类型的大额版（成倍灵石/高品丹药/稀有材料）
- `nothing` 一无所获

事件见闻拼成本次「探索纪要」进修行日志，形成叙事。事件权重随难度调整（越难越可能出 `enemy`/`fortune`）。

## 7. 时间与互斥（已定）

- **时长档**：沿用打坐的 **30 / 60 / 120 / 240 分钟**四档（白名单）。**难度越高，同一时长的收益越高**（收益系数含难度因子）。
- **挂机 + 惰性结算**：出发后进入倒计时，期满惰性结算——沿用打坐范式（`users` 补 `explore_*` 列 + DB 时钟守卫，见 §9），无需定时任务。
- **绝对互斥（已定）**：探索期间**不允许执行任何其他动作**——修炼、打坐、突破、再次探索**全部拦截**（各接口前置校验 + SQL 层守卫，与打坐同规，互相都要加对方的进行中守卫）。
- **单路**：同时只能有一次探索，不做多路/分身。

## 7.5 前端交互流程（已定：先投入后揭晓）

首页左侧导航「探索」按钮 → 打开探索弹窗 `components/ExploreModal.vue`：

1. **备程页**：展示四个时长档 + 一个「加持灵石」可选输入（默认 0，滑杆/输入框，标注投得越多收益越高、失败不退）。此时**尚未揭晓难度与区域**——保悬念，也防止玩家「看到危险区就不去」破坏 80/15/5 分布与风险设计。
2. 点「出发」→ 服务端掷难度 + 抽区域 + 扣加持灵石，返回区域简介（地名/难度标签/氛围文案，占位符已填道号/宗门/境界）→ 弹窗切到**探索中页**：区域简介 + 倒计时（无取消）。
3. 期间：境界卡的修炼/突破按钮、打坐入口均替换为禁用的「探索中 时:分:秒」（同打坐处理）。
4. 期满：倒计时归零自动结算（弹窗开则弹窗结算、闭则首页兜底，仿打坐）→ **纪要页**：逐条「探索见闻」（AI 事件文案）+ 收益汇总（灵石/丹药/材料/点缀）；若陨落则走**陨落结果页**（沉重文案 + 损失清单）。
5. 结算后全量刷新（profile/修炼/日志/榜单/今日统计）+ toast。

## 8. 与现有系统 / 经济平衡

- **不喧宾夺主**：修为主产出仍是修炼/打坐；探索的修为占比要小，主打灵石/丹药/奇遇。
- **灵石来源**：探索成为灵石主要「水龙头」，需与灵石消耗端（开宗立派 5000、后续商店/炼丹等）对齐，避免通胀。
- **丹药产出**：探索是丹药首个正式来源，掉率/品质需保守起步，后续可由炼丹系统补充。
- **今日统计 / 日志**：探索次数、探索收益可并入 `user_daily_stats`（可能需加列）与 `player_logs`（新增 type，如 `explore_start` / `explore` / 具体事件 type）。
- **宗门联动（后续）**：可扩展「宗门秘境 / 领地探索」，收益计入宗门活跃度——留作后期。

## 9. 数据结构设想（待定，仅抛砖）

**无静态区域配置表；有一张「区域池」表存预生成模板。** 可能涉及：

- **区域池表 `explore_area_pool`**（DeepSeek 预生成的模板）：`id, difficulty(simple/hard/danger), realm_band(境界段), name, intro, ambience, events(JSON 事件骨架), status(可用/已弃), created_time`，索引 `(difficulty, realm_band, status)`。抽取即按桶查、可标记短期降权防连刷。
- **进行中的探索存在 `users` 补列**（类比打坐三列）：`explore_start_time`、`explore_end_time`、`explore_difficulty`，外加 `explore_snapshot`（JSON，存本次「难度 + 抽中的区域模板 + 时长 + 加持灵石 + 预结算收益种子」）。**关键：出发时把难度与收益骨架定死存库**，期满只按快照结算——不在结算时再碰 AI，保证可复现、防利用。
- **材料表（已定要建）**：`materials`（`id, name, grade, realm, intro…`）+ `user_materials`（`user_id, material_id, quantity`，唯一键合并计数，仿 `user_pills`）。
- 结算函数 `node/src/utils/explore.js`（难度收益区间、事件掷骰、取整保底），控制器 `exploreController.js`（含惰性结算 `settleDueExplore`），路由 `GET/POST /api/user/explore`。
- **预生成服务**：`node/src/services/exploreGen.js`（或 `utils/deepseek.js`）封装 DeepSeek 调用——批量生成、超时/重试、JSON 校验、敏感词过滤、入池；由**后台手动触发**（管理页一个「补池」按钮）或定时任务跑，水位低时补池。
- **DeepSeek 配置（已定：后台绑定）**：不写死在代码/`.env`，改为**系统配置项**（`configs` 表，后台「系统配置」可改）——
  - `deepseek_api_key`（string，后台绑定）
  - `deepseek_model`（string，后台下拉选择，如 `deepseek-chat` / `deepseek-reasoner`）
  - `explore_pool_target_per_bucket`（number，每桶目标水位，初值 20）
  - 未配置 key 时预生成接口报错，探索仍可用（走内置模板兜底）。
- **出发接口入参**：`{ minutes, spendLingShi? }`（难度服务端掷，不接受客户端指定）；加持扣灵石在出发事务内完成。
- **日志/统计**：`player_logs` 新增 type `explore_start` / `explore`；`user_daily_stats` 可加列记探索次数/收益。

## 9.5 数值标定（初稿，均为可调初值）

> 以下数字是**调试初值**，落地后按经济观测再拧。全部由服务端结算，前端只展示。

**A. 基础灵石收益**（简单档、每小时；难度与时长为乘数）

| 境界段          | 简单档 灵石/小时 |
| --------------- | ---------------: |
| 凡俗 mortal     |               60 |
| 筑基 foundation |              260 |
| 元婴 nascent    |             1200 |
| 大乘 mahayana   |             5000 |
| 仙人 immortal   |            18000 |
| 仙王 sovereign  |            50000 |

- **难度乘数**：简单 ×1 / 困难 ×2.5 / 危险 ×6。
- **时长乘数**：按小时线性——30 分 ×0.5、60 分 ×1、120 分 ×2、240 分 ×4。
- **随机浮动**：最终灵石在算得值上 ±20% 抖动，四舍五入、保底 1。
- 公式：`灵石 = 段基础/时 × 时长小时 × 难度乘数 × (1±0.2) × (1 + 加持加成)`。
- 示例：筑基·困难·120 分 = 260 × 2 × 2.5 = **1300**（±20%）；再叠加持。

**B. 丹药/材料掉落**（每次探索独立掷，概率随难度升）

| 产出                             | 简单 | 困难 | 危险 |
| -------------------------------- | ---: | ---: | ---: |
| 成品丹（本段对应境界，凡品为主） |  25% |  40% |  55% |
| 成品丹升品（灵/道品）            |   — |   8% |  20% |
| 稀有炼丹材料（纯收集）           |   3% |  15% |  35% |

- 掉落的丹药/材料**境界匹配当前境界段**（服务端从 `pills`/`materials` 按段筛选后抽取）。
- 掉率受加持加成小幅提升（作用于上表概率，封顶不超过 +10 个百分点）。

**C. 修为/道韵点缀**（刻意压低，不抢主线）

- 每次探索额外给「当前境界 `advance_exp` 的 1%~3%」修为（随机），**单次不超过一次修炼收益（5%）的一半**。
- `advance_exp=0` 的仙人/仙王段：改给极小额**道韵**（如 1~3 点），不给修为。

**D. 奇遇（fortune 事件）**——低概率触发（简单 1% / 困难 3% / 危险 6%），一期发「已有类型大额版」：本次灵石收益 ×3~×5，或直接给一颗道品成品丹，或一份稀有材料（三选一随机）。

**E. 风险数值**（见 §5）：危险档陨落率 8%（段内 6%~12%）；困难档折损概率 30%。
