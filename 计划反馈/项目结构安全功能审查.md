## 总体结论

当前项目已经形成较稳定的前后端分离结构：`node/` 负责接口、鉴权、数据库与核心游戏规则，`web/` 负责 Vue 页面展示和交互，`资源文件/` 保存机制与丹药数据，`计划反馈/` 保存审查与规划文档。

本轮复查后，后端健康检查正常，前端生产构建通过，关键后端文件语法检查通过。生产配置校验、CORS 收敛、退出后 token 失效、打坐期间禁止修炼/突破等上轮高风险项已基本补齐。当前项目可以作为继续开发的稳定基础。

## 项目结构状态

- `node/src/server.js`：后端启动入口，已在启动前调用生产配置校验。
- `node/src/app.js`：Express 应用，统一挂载 `/api`，并按 `FRONTEND_ORIGIN` 控制 CORS。
- `node/src/routes/`：包含 `auth`、`user`、`admin` 三类路由。
- `node/src/controllers/`：包含登录注册、签到、修炼、打坐、突破、丹药和后台管理逻辑。
- `node/src/models/`：MySQL 数据访问层，核心玩家状态更新集中在 `userModel.js`。
- `web/src/views/`：前台首页、登录注册和后台页面。
- `web/src/components/`：签到、突破、打坐等弹窗组件。
- `资源文件/`：机制文档、境界数据、丹药数据。

需要注意：丹药数据同时存在于 `资源文件/丹药相关/pills.json` 和 `node/src/data/pills.json`，后端播种只读取 `node/src/data/pills.json`。后续应明确以哪份为主，避免数据分叉。

## 安全性复查

### 已稳定

- 生产环境会拒绝默认 `JWT_SECRET`、默认管理员密码和空数据库密码。
- `FRONTEND_ORIGIN` 已用于限制生产环境跨域来源；未配置时生产环境不开放跨域。
- 登录 token 会写入 `users.access_token`，鉴权时会比对数据库中的最新 token。
- 退出登录会清空 `access_token`，旧 token 随即失效。
- 管理端接口统一经过 `adminAuthRequired`，并校验 JWT role、数据库 role 与账号状态。

### 仍需关注

- 前端仍使用 `localStorage` 保存 token。短期可接受，但需要持续避免 XSS；长期建议迁移到 HttpOnly Cookie + CSRF 防护。
- 登录、注册接口暂未做限流或失败次数控制，生产环境建议增加 IP/账号维度限流。
- 管理后台拥有较强的数据修改能力，后续上线前建议补操作日志，记录管理员、目标对象、修改前后值和时间。

## 核心功能复查

### 签到

状态：基本可用。

已实现 24 小时冷却、后台开关 `sign_in_enabled`、按当前境界晋级需求发放修为/道韵/道法奖励，并使用原子 `UPDATE` 防重复签到。签到接口会先结算到期打坐，避免玩家离线收益长期不入账。

风险：如果打坐结算由签到接口抢先完成，前端不一定拿到 `settled` 结果展示，需要优化首页请求顺序或统一返回结算信息。

### 修炼

状态：基本可用。

已实现冷却配置 `cultivate_cooldown_seconds`、单次收益计算、圆满后禁止继续修炼、打坐期间禁止修炼。SQL 层也加入了打坐状态守卫，降低并发冲突。

未完成：修为丹、临时加成和其他丹药效果尚未接入修炼结算。

### 打坐

状态：功能闭环基本完成。

已实现 `GET /api/user/meditation` 与 `POST /api/user/meditation`，支持固定时长、倒计时、到期惰性结算、收益封顶到当前境界圆满，以及前端弹窗展示。打坐期间不能修炼、突破或再次打坐。

风险：打坐结算先在控制器计算剩余可增长空间，再执行 SQL 加修为。如果其他接口在两步之间增加修为，理论上可能超过 `advance_exp`。建议后续在 SQL 层使用 `LEAST()` 封顶，或引入事务锁。

### 突破

状态：基本可用。

已实现按境界需求判断修为/道韵/道法条件、雷劫死亡率、成功晋级、失败死亡次数和资源折损。打坐期间禁止突破，并已在 SQL 层加守卫。

## 当前优先级

1. 修正首页初始化顺序：优先请求打坐状态并处理 `settled`，再请求签到和修炼状态。
2. 打坐结算 SQL 层封顶，避免并发导致修为超过当前境界上限。
3. 登录/注册增加限流。
4. 明确丹药数据唯一来源，并规划玩家服丹和临时效果表。
5. 后续补测试框架，优先覆盖签到重复提交、修炼冷却、打坐结算、突破成功/失败。
