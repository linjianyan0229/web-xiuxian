# 文字修仙

**一款水墨仙侠风的网页文字修仙小游戏（前后端分离）**

修炼 · 打坐 · 渡劫突破 · 丹药 · 宗门 · 世界频道

</div>

---

## 项目简介

玩家注册后以「道号」行走此界，通过**修炼、打坐、每日签到**积累修为，冲击 83 阶境界体系（凡人 → 圣人），突破途中需直面**雷劫身陨**的风险；可开宗立派、收纳丹药、与众道友在世界频道论道。配套完整的后台管理端（用户 / 境界 / 丹药 / 宗门 / 系统配置）。

- `node/` — 后端：Node.js + Express 5（ESM）+ MySQL，原生 SQL 无 ORM
- `web/` — 前端：Vue 3 + Vite，水墨浅色主题 HUD

## 功能特性

### 游戏玩法
- **修炼**：点击修炼得当前境界圆满修为的 5%，冷却调息（后台可配）
- **打坐**：择时辰（一炷香～两个时辰）定时挂机，期间不可中断，离线到期惰性结算
- **境界突破**：按境界要求（经验 / 道韵 / 道法）判定，含雷劫的境界按死亡率掷骰——陨落则资源折半、死亡次数 +1
- **每日签到**：按境界随机区间发放修为 / 道韵 / 道法
- **丹药背包**：475 种丹药 × 凡/灵/道三档品质，支持筛选、赠送（事务转移）、丢弃
- **宗门**：耗 5000 灵石开宗立派，列表搜索筛选、详情（宗主 / 人数 / 活跃度）
- **世界频道**：全服聊天（增量轮询），内置约 5 万词敏感词过滤（含变体识别）
- **排行榜**：境界 / 在线 / 死亡三榜，在线榜实时显示玩家网络状态（心跳延迟 / 掉线）
- **角色**：性别立绘（注册页视频展示）、头像上传或外链、修行日志、今日修炼统计

### 账号与安全
- 注册**邮箱验证码**（SMTP 可配，未配置时开发环境验证码打印控制台）、**重置密码**
- JWT **单会话**：异地登录挤下线、登出 / 改密即吊销旧令牌
- 头像上传魔数复核（防伪造 MIME）、生产环境启动强校验密钥口令、CORS 白名单收敛

### 后台管理
仪表盘（统计卡 + 三榜）、用户管理（建号 / 编辑 / 禁用 / 删除级联）、境界列表、丹药管理、宗门管理、系统配置（可视化开关与数值）。

## 快速开始

```bash
# 1. 配置后端环境变量
cd node
cp .env.example .env        # 按需修改 MySQL 连接等（SMTP 可留空，开发环境验证码打印到控制台）

# 2. 启动后端（端口 3000；首次启动自动建库建表、导入境界/丹药数据、生成默认管理员）
npm install
npm run dev

# 3. 启动前端（端口 5173，/api 自动代理到 3000）
cd ../web
npm install
npm run dev
```

- 健康检查：`GET http://localhost:3000/api/health`
- 默认管理员：`admin / admin123456`（生产环境必须改，否则拒绝启动）；玩家与管理员统一在 `/login` 登录，按角色分流

## Docker 部署

提供 `docker-compose.yml` 一键编排 MySQL + 后端 + 前端（nginx）：

```bash
cp .env.docker.example .env   # 改掉密码与密钥
docker compose up -d --build
```

详细步骤见 [Docker部署教程.md](./Docker部署教程.md)。

## 设计原则

- **数值判定全在后端**：修炼收益、突破掷骰、冷却校验一律服务端结算，前端只做展示
- **并发安全靠原子 UPDATE**：资源变更的 WHERE 带全部前置守卫，`affectedRows=0` 即竞态失败
- **无 WebSocket / 定时任务**：聊天走增量轮询，打坐到期靠惰性结算，架构无状态易扩展
