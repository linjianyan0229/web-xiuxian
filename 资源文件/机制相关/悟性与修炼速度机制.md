# 悟性与修炼速度机制（暂定）

> **状态：暂定草案**，数值随后续功法/心法系统实装可调。
> **适用范围：功法、心法的修炼速度。**
> **明确排除：修为（cultivation）的获取不基于本机制**——打坐、签到等修为来源不乘修炼速度。

## 1. 背景

悟性（`users.comprehension`）已落库：整数百分比，上限 100；注册/后台建号时随机生成（99.5% 均匀取 1~10，0.5% 天纵奇才直接得 15）。本机制定义悟性如何转化为功法与心法的修炼速度——悟性越高，修炼速度越快。

## 2. 核心公式

所有玩家天生具有 **10% 的基础修炼速度**；悟性在此之上提供加成：

```
悟性加成       = 基础10% × 悟性%
最终修炼速度   = 基础10% + 悟性加成
             = 10% × (1 + 悟性 / 100)
```

- 下限：悟性 0（理论值）→ 10%（即只有基础速度）
- 上限：悟性 100（满值）→ 20%（恰好是基础速度的两倍）

## 3. 数值示例

| 悟性 | 悟性加成 | 最终修炼速度 | 备注 |
| ---: | ---: | ---: | --- |
| 1 | 0.1% | 10.1% | 初始下限 |
| 5 | 0.5% | 10.5% | |
| 10 | 1.0% | 11.0% | 初始上限 |
| 15 | 1.5% | 11.5% | 天纵奇才（0.5% 概率） |
| 30 | 3.0% | 13.0% | |
| 50 | 5.0% | 15.0% | |
| 100 | 10.0% | 20.0% | 悟性封顶 |

初始玩家的修炼速度落在 **10.1% ~ 11%**，天纵奇才为 **11.5%**。

## 4. 结算方式（暂定）

功法/心法系统尚未实装，暂定结算模型：每部功法/心法有自己的修炼进度（熟练度），每个结算周期（如一次打坐、一小时挂机）：

```
进度增量 = 该功法基础进度值 × 最终修炼速度
```

示例：某功法基础进度值为 1000 点/次，悟性 10 的玩家每次获得 `1000 × 11% = 110` 点；天纵奇才获得 115 点。基础进度值由功法品阶决定，待功法系统文档另行定义。

## 5. 边界与约束

- 悟性经 `clampComprehension()` 收敛到 0~100，因此修炼速度天然封顶 20%；未来丹药/奇遇提升悟性也不会突破。
- **修为不适用**：现有的签到修为奖励、以及未来打坐等修为获取，均不乘本速度。
- 数值结算一律在后端完成（防客户端篡改），前端只做展示。

## 6. 后端接入点

- `node/src/utils/comprehension.js` 已提供：
  - `BASE_CULTIVATION_SPEED_PERCENT = 10` — 基础修炼速度（%）
  - `cultivationSpeedPercent(comprehension)` — 返回最终修炼速度百分数（如悟性 15 → `11.5`）
- 未来功法/心法模块在后端结算进度时调用；前端 HUD「修炼增益」栏可用同一接口返回值展示。

## 7. 待定事项

- 功法品阶 → 基础进度值对照表（待功法系统设计）。
- 心法与功法是否共用同一速度，还是各自独立系数。
- 境界、丹药等其他修炼速度加成来源与悟性加成的叠加方式（相加还是相乘）。
