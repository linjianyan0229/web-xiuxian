# 宗门机制（列表 / 创建 / 详情）

> **状态：部分实装（2026-07）**——列表、搜索筛选、创建（扣灵石）、详情、后台管理已打通；加入/退出、成员管理、活跃度产出等玩法未实装（进度清单见 `计划反馈/宗门系统进度.md`）。
> **适用范围**：玩家浏览天下宗门、耗费灵石开宗立派并自任宗主；管理员在后台维护宗门。
> **与灵石的关系**：创建宗门是当前唯一的灵石消耗点；玩家暂无正式的灵石获取途径（待签到/挂机/历炼等产出接入），测试期由后台「用户编辑」发放。

## 1. 核心玩法

1. 首页左侧导航「宗门」进入**宗门列表页**（独立路由页 `/sect`，非弹窗），可搜索、筛选、翻页。
2. 「开宗立派」按钮打开创建弹窗：填**宗门名称 / 入门境界要求 / 宗门头像 / 宗门背景 / 宗门简介**，确认后**扣除灵石 5000**，创建者自任宗主并入驻本宗。
3. 点击任意宗门行弹出**详情**：背景横幅 + 宗徽、宗门名称、宗主（道号+境界）、宗门人数、活跃度、创建时间、入门要求、简介。
4. 一人只可身在一宗：已有宗门者不可再立派（加入他宗的玩法未实装）。
5. 管理员后台「宗门管理」：搜索/分页、编辑（名称/简介/活跃度）、解散（清空全体成员归属）。

## 2. 创建规则

| 项 | 规则 | 违例响应 |
| --- | --- | --- |
| 费用 | 固定 **5000 灵石**（`SECT_CREATE_COST`，硬编码于 `controllers/sectController.js`，后续可挪系统配置） | 灵石不足 409 |
| 名称 | 2~16 字符，全服唯一 | 过短/过长 400，重名 409 |
| 归属 | 创建者必须无宗门（`users.sect_id IS NULL`） | 已有宗门 409 |
| 境界要求 | 空=无要求；否则必须是 realms 表中的**大境界名**（凡人/炼体/…，共 26 档） | 非法 400 |
| 头像/背景 | 暂只支持外链 URL（http/https、≤255 字符，可空；文件上传未实装） | 格式不符 400 |
| 简介 | ≤500 字，可空 | 超长 400 |

- 创建成功写修行日志 type=`sect_create`（「散尽 5000 灵石，开宗立派…」）。
- 头像空时前端以宗门名首字作圆形宗徽（复用 `UserAvatar`）；背景空时详情用默认洞府图 `web/image/dongfu.webp`。

## 3. 境界要求的表示

- `sects.realm_req`：大境界**名称**（如「炼体」），空串=无要求——展示与筛选直接用名称，免连表。
- `sects.realm_req_rank`：该大境界**最小的 realms.id**（如炼体=2），0=无要求——为「加入宗门」预留的数值化门槛：加入时校验 `玩家 realm_id >= realm_req_rank` 即可（**加入玩法未实装，字段已就绪**）。
- 可选项由 `sectRealmOptions()` 提供：`SELECT realm, MIN(id) AS realm_rank FROM realms GROUP BY realm ORDER BY realm_rank`。
  ⚠ `rank` 是 MySQL 8 保留字，别名一律用 `realm_rank`。

## 4. 数据与并发

- **`sects` 表**：`id, name(唯一键), avatar, background, intro, realm_req, realm_req_rank, leader_id(宗主 users.id), activity(活跃度, 默认0), created_time`。
- **成员关系记在 `users.sect_id`**（NULL=散修），**无独立成员表**；宗门人数实时聚合 `(SELECT COUNT(*) FROM users WHERE sect_id = s.id)`。日后需要职位（长老/执事）时再引入 `sect_members` 表。
- **创建走事务**（`sectModel.createSect`）：
  1. 原子扣款：`UPDATE users SET ling_shi = ling_shi - 5000 WHERE id=? AND role=0 AND status=1 AND ling_shi>=5000 AND sect_id IS NULL`——余额与"无宗门"守卫合在一条 UPDATE 上，防并发双开/透支；affectedRows=0 即回滚返回 409。
  2. `INSERT sects`——重名撞唯一键抛 `ER_DUP_ENTRY`，回滚后译为 409。
  3. `UPDATE users SET sect_id = <新宗门id>`，提交。
- **解散走事务**（`disbandSect`）：清空全体成员 `sect_id` → 删宗门行。
- **删除玩家级联**：`userModel.deleteUser` 先调 `disbandSectsLedBy(userId)` 解散其执掌的宗门（须在删人前做，宗主关系尚在），普通成员随本人行删除自然脱离。

## 5. 活跃度（占位）

- 字段已建（`sects.activity`，默认 0），列表按 `activity DESC, id ASC` 排序，前后台均已展示。
- **产出机制未接入**：暂只能由后台「宗门管理 → 编辑」手工调整。
- 规划方向：按成员的签到/修炼/打坐行为累计（可复用 `user_daily_stats` 的挂点思路），按日或按周衰减。

## 6. 后端接入点

- `node/src/models/sectModel.js`：`listSects()`（分页+名称搜索+境界要求筛选，`realmReq='none'` 只看无要求）、`findSectById()`、`sectRealmOptions()`/`findRealmOption()`、`createSect()`（事务）、`updateSect()`（白名单）、`disbandSect()`（事务）、`disbandSectsLedBy()`。
- `node/src/controllers/sectController.js`：前台 `getSects`/`getSectMeta`/`getSectDetail`/`doCreateSect` + 后台 `adminGetSects`/`adminEditSect`/`adminDisbandSect`，`SECT_CREATE_COST = 5000`。
- 路由（前台，`/sects/meta` 须先于 `/sects/:id` 声明）：
  - `GET /api/user/sects?page&pageSize&keyword&realmReq` → `{ list, total, page, pageSize, mySectId }`
  - `GET /api/user/sects/meta` → `{ realms:[{realm, realm_rank}], createCost }`
  - `GET /api/user/sects/:id` → `{ sect }`
  - `POST /api/user/sects` → `{ sect, user }`（user 含扣减后灵石与 sect_id）
- 路由（后台）：`GET /api/admin/sects`、`PUT /api/admin/sects/:id`（name/intro/activity）、`DELETE /api/admin/sects/:id`。
- 修行日志 `player_logs`：`sect_create`。

## 7. 前端

- **列表页** `web/src/views/SectView.vue`（路由 `/sect`，`GAME_PAGES` 成员，与首页互切有方向滑动过渡）：dongfu.webp 背景 + 宣纸轻纱；与 HUD 同规**锁一屏、列表内滚**（防切页时滚动条出没引发宽高跳动）；顶栏「返回洞府」、灵石余额；工具栏搜索/境界要求筛选/「开宗立派」；本宗行标「本宗」（接口 `mySectId`）。
- **创建弹窗** `components/SectCreateModal.vue`：五字段 + 费用提示（现有灵石对照），成功回抛 `{ sect, user }` 由页面落登录态并刷新列表。
- **详情弹窗** `components/SectDetailModal.vue`：打开时按 id 拉最新详情（人数/活跃度实时）；背景横幅压渐变 + 悬浮宗徽。
- **后台** `views/admin/SectsView.vue`：列表/搜索/分页 + 编辑弹窗 + 解散二次确认（提示成员数、不可撤销）。
